# [서비스 기획서] 토익넷 2.0 (TOEICNET 2.0) - 1단계 MVP

## 1. 프로젝트 개요

- **서비스명**: 토익넷 2.0 (TOEICNET 2.0) - AI Bot 대전 에디션
- **한줄 정의**: 실시간 매칭 부담 없이 컴퓨터(AI)와 1:1 속도전을 펼치는 캐주얼 토익 게이미피케이션 웹 앱

### 1단계 목표

1. 네트워크 동기화, 동시접속자 확보 등의 백엔드 리스크를 배제하고 '시간 압박 속에서 문제를 푸는 쫄깃함'이라는 핵심 UX 검증.
2. LLM 프롬프트 파이프라인을 구축하여 문제 수급 및 콘텐츠 공급 자동화 체계 테스트.

---

## 2. 핵심 게임 메커니즘 (Core Gameplay)

### ① AI Bot 1:1 대전 프로그레스

- 유저는 방 입장과 동시에 대기 시간 없이 즉시 지정된 난이도의 컴퓨터(AI Bot)와 1:1 대전을 시작합니다.
- 화면에 '컴퓨터의 문제 풀이 게이지(Bot Progress Bar)'가 실시간으로 차오르는 모습을 시각화하여, 유저에게 실제 사람과 대전하는 듯한 시간적 압박감과 몰입감을 부여합니다.

### ② 스코어링 시스템 (정답 + 속도 보너스)

- **기본 점수**: 문제를 맞히면 기본 100점 획득.
- **속도 보너스**: 문제를 맞힌 시점에 남은 제한시간(초)을 보너스 점수로 합산. (빨리 맞힐수록 고득점)
- **컴퓨터 채점**: 컴퓨터는 설정된 난이도(확률 및 속도)에 따라 유저와 별개로 백그라운드 타이머를 돌려 정답/오답 여부를 결정하고 점수를 가져갑니다.

### ③ 게임 루프 및 문항 구성

- **문항 수**: 한 판당 토익 RC Part 5(단문 공란 메우기) 위주의 10문항으로 구성하여 판당 2~3분 내외의 빠른 템포 유지.
- **종료 조건**: 10문항을 모두 풀면 즉시 최종 스코어보드가 노출되며 승/패 결과 결정.

---

## 3. 난이도 및 밸런스 설계

유저의 수준에 맞춰 컴퓨터의 풀이 속도(타이머 제한)와 정답 확률을 가변적으로 조절합니다.

| 난이도 (Difficulty) | 컴퓨터 정답률 (Accuracy) | 컴퓨터 문제 풀이 소요 시간 (Speed) | 기대 효과 |
|---|---|---|---|
| EASY (초급) | 50% | 5.0초 ~ 8.0초 (무작위) | 토익 입문자도 쉽게 승리하며 성취감 고취 |
| MEDIUM (중급) | 70% | 3.5초 ~ 6.0초 (무작위) | 일반적인 유저가 가장 긴장감을 느낄 수 있는 밸런스 |
| HARD (고급) | 85% | 2.0초 ~ 4.0초 (무작위) | 고득점자를 위한 모드로, 극도의 속도전 유도 |

---

## 4. MVP 아키텍처 및 기술 스택

프론트엔드 중심의 경량 아키텍처로, 서버 비용을 최소화하고 브라우저에서 즉시 실행 가능하도록 구성합니다.

```
[Front-end: Next.js (React) + Tailwind CSS]
       │
       ├─> (인게임 타이머 및 AI 봇 행동 시뮬레이션 로직 구동)
       │
[AI Layer / DB: Supabase + LLM API (OpenAI/Anthropic)]
       └─> JSON 기반 문제 은행 데이터 수급 및 저장
```

- **Front-end**: Next.js, Tailwind CSS (Responsive Web 디자인으로 모바일/PC 완벽 지원)
- **Back-end/DB**: Supabase (출제될 토익 문제 JSON 데이터를 담아두는 심플한 Table 1개로 운영)
- **AI Engine**: Claude 3.5 Sonnet 혹은 GPT-4o API
  - **역할**: 지정된 JSON 스키마에 맞춰 Part 5 유형의 문제와 정답, 해설을 자동으로 대량 생성하여 데이터베이스에 적재.

## 5. 1단계 MVP 화면 구조 (Wireframe 요약)

### 메인 화면 (Lobby)

- 게임 시작 버튼 및 컴퓨터 난이도 선택 라디오 버튼 (EASY / MEDIUM / HARD).

### 게임 화면 (In-Game Room)

- **상단**: 내 점수 vs 컴퓨터 점수 현황 및 컴퓨터의 실시간 게이지 바.
- **중앙**: 토익 RC Part 5 문제 텍스트 지문 및 남은 시간 카운트다운 타이머.
- **하단**: 네 개의 선지 (A), (B), (C), (D) 선택 버튼.

### 결과 화면 (Result Page)

- 최종 스코어 비교를 통한 승/패 연출.
- 10개 문항에 대한 정답 유무 피드백 및 AI가 생성한 오답 해설 리포트 노출.

---

## 6. 개발 및 검증 로드맵 (4주 완성)

- **1~2주차 [콘텐츠 및 DB 세팅]**: LLM 프롬프트를 통해 퀄리티 높은 토익 Part 5 문제 200~300개를 추출하여 Supabase DB에 적재.
- **3주차 [핵심 로직 개발]**: Next.js 환경에서 유저 타이머와 컴퓨터(Bot)의 게이지 타이머가 동시에 시뮬레이션되는 핵심 인게임 룸 컴포넌트 개발.
- **4주차 [UI 다듬기 및 테스트]**: Tailwind CSS로 게임 오락실 느낌의 디자인 요소(폰트, 컬러, 효과음 등)를 반영하고 지인들을 대상으로 타임 어택 몰입도 피드백 수렴.

---

이 방향대로라면 복잡한 실시간 매칭 서버를 지을 필요가 없기 때문에 혼자서도 몇 주 안에 충분히 돌아가는 프로토타입을 완성할 수 있습니다.
