## 1. 목적 / 한 줄 정의

토익 RC 문제를, **남은 시간을 다투는 AI 챌린저와의 1:1 속도전**으로 풀어
"혼자 푸는 연습"을 "이겨야 하는 한 판"으로 바꾸는 게이미피케이션 학습 모드.

### 성공 지표 (가설)
- 1세션당 평균 플레이 판 수 ↑ (리매치율 ≥ 30%)
- 판당 완주율(10문항 끝까지) ≥ 80%
- 재방문(D1 retention) ↑ — 크레딧/랭킹이 복귀 동기

## 2. 현재 상태 vs 목표 (Gap)

| 영역 | 현재(`src/game/store.ts`) | 목표(대결 모드) |
|---|---|---|
| 진행 방식 | **무한 연습** (queue 무한 리필) | **고정 10문항 1판**, 종료 시 승패 결정 |
| 상대 | 없음 | **AI 챌린저** (난이도별 정답률·풀이속도 시뮬) |
| 점수 | `solved/correct/streak` 카운트만 | **정답(기본점) + 속도 보너스** 합산, 양측 비교 |
| 시간 | 제한 없음 | **문항별 카운트다운**(예: 20초) |
| 결과 | 복습 리포트(`result/page.tsx`) | **GAME RESULT**(승패·랭킹·크레딧·진행현황·리매치) |
| 재화 | 없음 | **크레딧**(승리/미션 보상, 별명·플레이어 ID) |
| 화면 흐름 | 로비→연습→결과 | 로비→**방개설/매칭**→**인게임 대결**→**결과(리매치)** |

> 연습 모드는 유지(학습용)하되, 대결 모드를 **별도 진입 경로**로 추가한다.
> 가능하면 문항 풀/필터 로직(`buildPool`, 콘텐츠 158세트)은 **재사용**한다.

## 3. 핵심 대결 메커니즘

### 3-1. 한 판 구성
- **문항 수**: 1판 = RC 10문항 (기획서 기준). 목업의 "게임진행현황"도 문항별 칸 그리드.
- **문항 출처**: 기존 `content/sets` 은행에서 선택 파트/난이도로 10문항 추출(셔플).
- **종료 조건**: 10문항 모두 풀면 즉시 결과 화면 → 승/패 확정.

### 3-2. 스코어링
- **기본 점수**: 정답 1문항 = **100점**.
- **속도 보너스**: 정답 시점의 **남은 제한시간(초)**을 가산 (빨리 맞힐수록 ↑).
- **오답**: 0점 (감점 없음, MVP).
- **최종 점수** = Σ(기본 + 보너스). 양측 점수로 1위/2위(RANKING) 결정.

### 3-3. AI 챌린저 시뮬레이션 (서버/실시간 매칭 없음, 프론트 시뮬)
- 유저 타이머와 **독립된 백그라운드 타이머**로 봇이 각 문항을 "푼다".
- 난이도별 **정답 확률 + 풀이 소요시간(랜덤 범위)** 으로 정답/오답·점수 결정.
- 화면에는 **봇 진행 게이지(Bot Progress Bar)** 가 실시간으로 차오름 → 압박감.

| 난이도 | 봇 정답률 | 봇 문항 풀이시간 | 기대 효과 |
|---|---|---|---|
| EASY (초급) | 50% | 5.0~8.0초 (랜덤) | 입문자 승리 경험 |
| MEDIUM (중급) | 70% | 3.5~6.0초 (랜덤) | 가장 긴장되는 밸런스 |
| HARD (고급) | 85% | 2.0~4.0초 (랜덤) | 극한 속도전 |

> 봇의 속도 보너스: 봇이 "맞힌" 문항도 동일 공식(남은 시간 환산)으로 점수화해
> 유저 점수와 같은 척도로 비교한다.

## 4. 게임 루프 & 화면 흐름

```
[로비/방개설]  →  [매칭 연출(3·2·1)]  →  [인게임 대결 ×10]  →  [GAME RESULT]
   파트/난이도        AI CHALLENGER         문제+타이머+봇게이지       승패·크레딧
   선택, 방개설         입장 연출            정답→다음 문항             리매치/리뷰/방개설
                                                                      ↑__리매치(1분)__|
```

### 4-1. 로비 / 방개설
- 파트(5/6/7) · 난이도(EASY/MEDIUM/HARD) 선택 후 **[방개설]** → 즉시 AI 매칭(대기 0).
- 유저 식별: **별명(닉네임) + 플레이어 ID**(예: `P2479`) — 로컬 저장(MVP, 계정 없음).

### 4-2. 인게임 대결 (신규 화면)
- **상단**: 내 점수 vs AI 점수, **AI 진행 게이지 바**, 현재 문항 번호(예: 4/10).
- **중앙**: 문제 지문/문장 + **남은 시간 카운트다운**.
- **하단**: 보기 (A)(B)(C)(D).
- 정답 선택 즉시: 정/오 피드백 → 점수 가산 → 다음 문항. (10문항 후 결과)

### 4-3. 결과 — GAME RESULT (목업 정밀 반영, §5)

## 5. 결과 화면 명세 (목업 컴포넌트 분해)

상단 좌측 **GAME RESULT 표**:

| 컬럼 | 내용 | MVP 처리 |
|---|---|---|
| 게임참가자 | 아바타 + 별명 (유저 / `AI CHALLENGER`) | ✅ |
| **L/C** | Listening 점수 (원형 게이지) | ⚠ **향후** — RC 전용이라 비활성/placeholder |
| **R/C** | Reading 점수 (원형 게이지) | ✅ 핵심 지표 |
| RANKING | 1위/2위 | ✅ 점수 비교 |
| CREDIT | 이번 판 획득 크레딧 | ✅ |
| REMATCH | 리매치 아이콘(유저 행) | ✅ |

- **우측 프로필 카드**: 아바타, `별명: …`, `플레이어 ID: P….`
- **좌하단 결과 배너**: "○○ 님은 N등으로 '승리/패배' 하셨습니다." + 💲크레딧,
  `Mission Complete : N크레딧 추가 획득!` + 달성 배지.
- **중앙 버튼 3종**:
  - `틀린문제 REVIEW` — 오답 복습(기존 `ReviewReport` 재사용). **시작 시 자동 퇴실**.
  - `REMATCH` — 같은 조건 재대결. **종료 후 1분 내에만** 가능.
  - `방개설` — 새 조건으로 방 생성(로비 복귀).
- **우하단 게임진행현황**: 참가자별 **문항 결과 그리드**(칸당 C, 정답=초록 / 오답=빨강).
  유저 행·AI 행 각각 10칸.

### 5-1. 시스템 메시지 (목업 하단 그대로 규칙화)
- `REMATCH는 게임 종료 후 1분 안에만 가능합니다.`
- `Review를 시작하시면 자동으로 퇴실 됩니다.` (리뷰 진입 = 방 종료)
- `10초 후에 REMATCH 권한이 사라집니다.` (1분 카운트다운 막바지 경고)
- `시간이 초과되어 REMATCH 권한이 소멸 되었습니다.` (만료 후 버튼 비활성)

## 7. 데이터 / 상태 모델 (구현 가이드)

기존 `usePracticeStore`를 건드리지 않고 **별도 `useMatchStore`** 신설 권장(연습 모드 보존).

```ts
type Participant = "user" | "ai";

interface MatchPlayer {
  name: string;            // 별명
  playerId: string;        // P2479
  score: number;           // 기본+보너스 누적 (R/C)
  results: boolean[];      // 문항별 정/오 (게임진행현황 그리드)
  rank?: 1 | 2;
}

interface MatchState {
  status: "lobby" | "countdown" | "playing" | "result";
  part: Part; difficulty: Difficulty;
  questions: PassageQuestion[];   // 이번 판 10문항 (기존 buildPool 재사용)
  qIndex: number;                  // 0..9
  perQuestionSeconds: number;      // 예: 20
  remaining: number;               // 현재 문항 남은 시간
  user: MatchPlayer; ai: MatchPlayer;
  aiProgress: number;              // 0..1 봇 게이지
  rematchDeadline?: number;        // 종료시각+60s (epoch ms)
  credits: number;                 // 누적 크레딧
  // actions: startMatch, answer, tickTimer, tickAi, finish, rematch, exit ...
}
```

- 봇 시뮬: 문항 시작 시 `[min,max]`에서 풀이시간 t·정답여부(p) 추첨 → `t`초 뒤
  봇 결과 확정 + 게이지는 `tickAi`로 `t`에 맞춰 보간.
- 크레딧/별명/플레이어ID는 `localStorage` 영속(`src/game/progress.ts` 패턴 참고).

### 재사용 가능한 기존 자산
- 문항 풀/필터: `buildPool`, `cycleFill` (`store.ts`) — 10개만 뽑도록 래핑.
- 콘텐츠 로딩: `/api/sets`, `loadAllSets` (158세트).
- 오답 리뷰: `src/components/result/ReviewReport.tsx`, `TypeBreakdown.tsx`.
- 점수 애니메이션: `src/components/ui/AnimatedNumber.tsx`.

### 신규 컴포넌트(안)
- `src/app/match/page.tsx` (인게임 대결), `src/app/match/result/page.tsx`
- `components/match/` : `BotProgressBar`, `MatchHud`(점수/타이머), `CountdownIntro`,
  `GameResultTable`, `ProgressGrid`(C 칸), `ProfileCard`, `RematchTimer`.

## 8. 범위 (MVP) / 비범위

**MVP 포함**
- RC(Part 5/6/7) 기반 1판 10문항 AI 대결, 속도 보너스, 승패/랭킹.
- GAME RESULT 화면(목업), 크레딧 획득·누적, 리매치(1분), 오답 리뷰 연결.
- 별명/플레이어ID 로컬 영속.

**MVP 비범위(향후)**
- **L/C(Listening)** 지표·콘텐츠 — 목업엔 있으나 현재 RC 전용. placeholder로 표기.
- 실시간 사람 vs 사람 매칭(서버 동기화) — 기획서대로 의도적 제외.
- 크레딧 소비처/상점, 글로벌 랭킹/리더보드, 계정·로그인.
- 효과음/햅틱 등 연출 고도화(2차).

## 9. 구현 마일스톤(제안)

1. **M1 — 대결 엔진**: `useMatchStore`(타이머·봇 시뮬·스코어), 10문항 추출.
2. **M2 — 인게임 화면**: `match/page.tsx` + HUD + 봇 게이지 + 카운트다운 인트로.
3. **M3 — 결과 화면**: GAME RESULT 표/진행현황 그리드/프로필/배너(목업 1:1).
4. **M4 — 크레딧·리매치**: 보상 규칙, 1분 리매치 타이머/시스템 메시지, 리뷰 연결.
5. **M5 — 다듬기**: 반응형(모바일/PC), 연출, 밸런스 튜닝.
